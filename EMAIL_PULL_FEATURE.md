# 邮箱推送功能集成说明

## 功能概述

在发票调试工作台的图片上传区域添加了 Tab 切换功能，支持两种发票录入方式：
1. **图片上传**：传统的本地文件上传方式
2. **邮箱拉取**：从 QQ 邮箱拉取发票邮件并自动识别

## 实现内容

### 1. Tab 切换界面

- 使用 Element UI 的 `el-tabs` 组件实现 Tab 切换
- 默认显示"图片上传"Tab
- 新增"邮箱拉取"Tab，包含完整的邮箱配置和处理流程

### 2. 邮箱拉取功能

#### 2.1 输入参数
- **邮箱地址**：支持 QQ 邮箱、163 邮箱等常见邮箱
- **授权码**：邮箱的 SMTP/IMAP 授权码（非登录密码）
- **时间范围**：可选最近 7 天、1 个月、3 个月、6 个月、12 个月

#### 2.2 统计展示
实时显示 4 个关键指标：
- **匹配邮件数**：符合"发票"主题的邮件数量
- **已下载附件**：成功下载的附件数量
- **已成功导入**：成功导入数据库的发票数量
- **失败数量**：处理失败的发票数量

#### 2.3 进度显示
- 实时进度条显示下载进度百分比
- 完成后显示"完成"状态

#### 2.4 处理日志
- 滚动显示实时处理日志
- 最多保留最近 100 条日志
- 每条日志带时间戳

### 3. 自动化流程

1. **启动拉取**：用户点击"开始拉取"按钮
2. **后台处理**：
   - 连接 IMAP 服务器
   - 筛选含"发票"主题的邮件
   - 提取附件或下载链接
   - 每下载一个附件立即导入数据库
   - 异步执行，不阻塞用户操作
3. **实时轮询**：每 2 秒查询一次任务状态
4. **自动刷新**：任务完成后自动刷新发票列表

### 4. 技术实现

#### 前端修改

**文件**: `fp_view/src/views/Invoice.vue`

**新增数据属性**:
```javascript
activeTab: 'upload',           // Tab 切换
emailForm: {                   // 邮箱配置
  mailbox: '',
  authCode: '',
  rangeKey: '3m',
},
emailPulling: false,           // 拉取状态
emailJobId: null,              // 任务 ID
emailStats: {                  // 统计数据
  matched_messages: 0,
  downloaded: 0,
  imported: 0,
  failed: 0,
},
emailLogs: [],                 // 日志数组
emailPollTimer: null,          // 轮询定时器
```

**新增方法**:
- `startEmailPush()`: 启动邮箱推送任务
- `pollEmailStatus()`: 轮询任务状态（2 秒间隔）
- `formatEmailProgress()`: 格式化进度显示
- `getEmailProgress()`: 计算进度百分比

**新增样式**:
- `.upload-tabs`: Tab 容器样式
- `.email-panel`: 邮箱面板样式
- `.email-stats`: 统计卡片样式
- `.email-progress`: 进度条样式
- `.email-logs`: 日志区域样式

#### 后端接口

**已有接口** (位于 `fp_api/routes.py`):

1. **启动邮箱推送**
   - 路径：`POST /api/email-push/{user_id}/start`
   - 参数：`range_key`, `mailbox`, `auth_code`
   - 返回：`{ job_id: "..." }`

2. **查询任务状态**
   - 路径：`GET /api/email-push/status/{job_id}`
   - 返回：
     ```json
     {
       "status": "running|completed|error",
       "matched_messages": 10,
       "downloaded": 8,
       "imported": 7,
       "failed": 1,
       "logs": ["...", "..."]
     }
     ```

**核心逻辑** (位于 `fp_api/email_push.py`):
- `start_email_push_job()`: 创建异步任务
- `_run_email_push_job_sync()`: 后台线程执行拉取
- 支持附件提取和链接下载
- 自动分组选择文件（避免重复）
- 调用 `file_service.import_local_file()` 导入数据库

## 使用说明

### 1. 切换到邮箱拉取 Tab

点击工作台上方的"邮箱拉取"标签

### 2. 填写邮箱信息

- 输入邮箱地址（如：`xxx@qq.com`）
- 输入授权码（需在邮箱设置中生成 IMAP 授权码）
- 选择时间范围（默认：最近 3 个月）

### 3. 开始拉取

点击"开始拉取"按钮，系统会：
1. 连接邮箱服务器
2. 筛选发票邮件
3. 下载附件和解析链接
4. 自动导入数据库并触发识别

### 4. 查看进度

- 统计卡片实时更新数字
- 进度条显示下载百分比
- 日志区域显示详细过程

### 5. 完成任务

任务完成后：
- 自动停止轮询
- 自动刷新发票列表
- 可切换到图片上传 Tab 继续其他操作

## 注意事项

### 邮箱授权码获取

以 QQ 邮箱为例：
1. 登录网页版 QQ 邮箱
2. 进入"设置" → "账户"
3. 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV 服务"
4. 开启"IMAP/SMTP 服务"
5. 生成授权码（发送到手机验证）

### 支持的邮箱类型

- **QQ 邮箱**：`imap.qq.com`
- **163 邮箱**：`imap.163.com`
- **126 邮箱**：`imap.126.com`
- **Gmail**：`imap.gmail.com`
- 其他邮箱：自动尝试 `imap.{域名}`

### 文件格式支持

支持的附件格式：
- 图片：`.jpg`, `.jpeg`, `.png`, `.webp`
- 文档：`.pdf`, `.ofd`, `.xml`

### 并发控制

- 后台异步线程处理，不阻塞前端
- 每 2 秒轮询一次状态
- 自动清理过期定时器（页面关闭时）

## 测试建议

### 测试场景 1：少量邮件
- 准备：发送 3-5 封含发票附件的邮件到测试邮箱
- 操作：选择最近 7 天范围，启动拉取
- 验证：统计数字准确，进度条正常，日志清晰

### 测试场景 2：大量邮件
- 准备：发送 20+ 封含发票的邮件
- 操作：选择最近 3 个月范围
- 验证：性能稳定，内存无泄漏，最终全部导入

### 测试场景 3：混合来源
- 准备：部分邮件带附件，部分带下载链接
- 操作：启动拉取
- 验证：两种方式都能正确处理

### 测试场景 4：错误处理
- 准备：错误的授权码
- 操作：启动拉取
- 验证：显示明确的错误提示
