# 项目结构说明 - 三个独立项目

## 📁 项目总览

你现在拥有**三个完全独立**的项目，每个都有不同的用途和技术栈：

```
fapiaohezi/
├── fp_api/          # 后端 API（Python FastAPI）
├── fp_admin/        # 后台管理系统（Vue2 + Element UI）
├── fp_view/         # Web 端用户界面（Vue2 + Element UI）
└── fp_app/          # 移动端 App（UniApp + Vue2）⭐ 新建
```

---

## 🔍 各项目的定位

### 1️⃣ **fp_api** - 后端服务
- **技术栈**: Python + FastAPI
- **作用**: 提供所有 API 接口
- **运行方式**: `python main_refactored.py`
- **端口**: 8000
- **功能**: 
  - 用户认证
  - 发票上传和 OCR 识别
  - 数据管理
  - 数据库操作

### 2️⃣ **fp_admin** - 后台管理系统
- **技术栈**: Vue 2 + Element UI
- **作用**: 管理员使用的全屏 Web 应用
- **运行方式**: `npm run serve`
- **端口**: 8081
- **访问地址**: http://localhost:8081
- **功能**:
  - 📊 Dashboard 数据统计图表
  - 👥 用户管理（查看所有用户、删除用户）
  - 📄 发票管理（查看所有发票、导出 Excel）
  - 🔐 管理员视角

### 3️⃣ **fp_view** - Web 端用户界面
- **技术站**: Vue 2 + Element UI
- **作用**: 普通用户使用的全屏 Web 应用
- **运行方式**: `npm run serve`
- **端口**: 8080（假设）
- **功能**:
  - 用户上传发票
  - 查看个人发票历史
  - 个人中心管理
  - 普通用户视角

### 4️⃣ **fp_app** - 移动端 App ⭐ 新建
- **技术栈**: UniApp + Vue 2
- **作用**: 移动端应用（可发布到多个平台）
- **运行方式**: 
  - HBuilderX: 运行到浏览器
  - CLI: `npm run dev:h5`
- **端口**: 8080（H5）
- **支持平台**:
  - ✅ H5 网页版
  - ✅ 微信小程序
  - ✅ Android App
  - ✅ iOS App
- **功能**:
  - 📷 拍照上传发票
  - 📱 移动端优化的界面
  - 📋 历史记录查看
  - 👤 个人中心
  - 🎨 简洁现代的 UI

---

## 🆚 fp_view vs fp_app 的区别

你之前可能混淆了这两个，现在明确说明：

| 特性 | fp_view | fp_app |
|------|---------|--------|
| **本质** | 传统 Web 应用 | 跨平台移动应用 |
| **技术栈** | Vue2 + Element UI | UniApp + Vue2 |
| **UI 框架** | Element UI 组件 | 原生组件/uni-ui |
| **单位** | px | rpx（响应式） |
| **导航** | Vue Router | TabBar |
| **编译** | webpack | uni-app 编译器 |
| **发布** | Web 服务器 | H5/小程序/App |
| **代码位置** | `src/views/` | `src/pages/` |
| **入口** | `main.js` + router | `main.js` + pages.json |
| **适用场景** | PC 浏览器 | 手机/平板/小程序 |

### 简单理解

- **fp_view** = 用 Element UI 做的网站，在电脑浏览器看
- **fp_app** = 用 UniApp 做的 App，可以在手机上看，也能编译成小程序

## 🎯 使用场景

### 场景 1：管理员查看数据
👉 打开 **fp_admin** (http://localhost:8081)
- 看统计图表
- 管理所有用户
- 查看所有发票

### 场景 2：PC 端用户上传发票
👉 打开 **fp_view** (Web 版)
- 在电脑上操作
- 大屏显示
- 鼠标操作

### 场景 3：手机拍照上传
👉 打开 **fp_app** (移动版)
- 直接拍照
- 随时随地上传
- 触屏操作
- 可以打包成 App 安装到手机

## 🚀 启动方式对比

### fp_admin（后台管理）
```bash
cd fp_admin
npm install
npm run serve
# 访问 http://localhost:8081
```

### fp_view（Web 用户版）
```bash
cd fp_view
npm install
npm run serve
# 访问 http://localhost:8080
```

### fp_app（移动端 App）
```bash
# 方式一：HBuilderX（推荐）
1. 打开 HBuilderX
2. 打开 fp_app 文件夹
3. 运行 > 运行到浏览器

# 方式二：CLI
cd fp_app
npm install
npm run dev:h5
# 访问 http://localhost:8080
```

## 💡 建议的使用方式

1. **后端必开**: `fp_api` 必须一直运行（端口 8000）

2. **开发前端**: 
   - 同时开 `fp_admin` 和 `fp_app`
   - 一个做管理后台
   - 一个做移动端展示

3. **fp_view 的处理**:
   - 如果要保留：可以改成其他用途
   - 如果不需要：可以删除或重命名

## 📊 数据流向

```
用户操作 → fp_app / fp_view / fp_admin
              ↓
         fp_api (端口 8000)
              ↓
          main.db 数据库
              ↓
        用户文件目录
```

所有前端项目都调用同一个后端 API！
